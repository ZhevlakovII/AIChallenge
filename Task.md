Нужно доработать существующий KMP-клиент.
Проект состоит из следующих модулей:

* **KMP Desktop клиент:** @composeApp
* **Общий модуль (common):** @shared\sharedOld
* **Бэкенд / MCP-сервер:** @instances\servers\mcp
* **RAG:** @rag\doc-indexer
В данный момент разрешается дорабатывать устаревшие модули.

## **Требуемые задачи**

### **1. Реализовать RAG для документации проекта**

Добавить в Desktop KMP клиент функциональность, позволяющую:

* указать пользователю **папку с документацией проекта** (README, API, схемы данных, ADR, любые .md/.txt файлы);
* встроить локальный RAG-пайплайн, который:

    * сканирует и индексирует документы;
    * хранит эмбеддинги в локальной БД в виде json-файла (реализовано отдельным CLI-приложением);
    * позволяет по запросу пользователя к проекту выдавать релевантные фрагменты документации;
    * обрабатывает вопросы в стиле:
      *«Как работает авторизация?»*, *«Покажи схему БД»*, *«Где лежат параметры для VPN?»*;
* интегрировать RAG в систему команд, чтобы ответы включали подсвеченные куски документации.

### **2. MCP-сервер с инструментами для Git**

Добавить на MCP-сервер новые инструменты:

* `git.current_branch` — возвращает текущую ветку;
* `git.last_commit` — SHA + сообщение последнего коммита;
* `git.commit_count` — количество коммитов в текущей ветке.

Инструменты должны быть доступны ассистенту через MCP протокол.

### **3. Команда `/help` для ассистента**

Создать команду `/help`, которая:

* отвечает на вопросы о проекте;
* использует RAG из задания №1;
* подсказывает:
    * фрагменты кода,
    * правила стиля,
    * архитектурные решения,
    * расположение файлов,
    * релевантные части README/API;
* автоматически вызывает MCP-инструменты, когда вопрос связан с git.
  Например:
  *«На какой мы ветке?»*,
  *«Покажи последний коммит».*
